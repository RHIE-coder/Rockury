# Code Review Prompt

## Usage
코드 리뷰 요청 시 사용하는 표준 프롬프트

---

## Standard Review

```
코드 리뷰를 수행해주세요.

## 검토 대상
[파일 경로 또는 변경 내용]

## 검토 항목
1. 코드 품질
   - 네이밍 컨벤션
   - 함수/클래스 설계
   - 중복 코드
   - 복잡도

2. 보안
   - 입력 검증
   - 인증/인가
   - 민감 정보 노출
   - SQL/XSS 취약점

3. 성능
   - N+1 쿼리
   - 불필요한 연산
   - 메모리 누수 가능성

4. 테스트
   - 커버리지 충분성
   - 엣지 케이스
   - 테스트 품질

## 응답 형식
### Summary
[전체 요약]

### Issues
- [심각도: Critical/High/Medium/Low] [설명]

### Suggestions
- [제안 사항]

### Good Points
- [잘된 점]
```

---

## Quick Review

```
빠른 코드 리뷰를 요청합니다.

## 변경 내용
[간단한 변경 설명]

## 파일
[파일 목록]

주요 문제점만 짚어주세요.
```

---

## Security-Focused Review

```
보안 관점에서 코드를 리뷰해주세요.

## 검토 대상
[파일 경로]

## 특히 확인해야 할 항목
- [ ] 입력 검증 (Zod 사용 여부)
- [ ] SQL Injection 방지
- [ ] XSS 방지
- [ ] 인증/인가 검증
- [ ] 민감 정보 처리
- [ ] 에러 메시지 노출
- [ ] Rate limiting
- [ ] CORS 설정
```

---

## Performance Review

```
성능 관점에서 코드를 리뷰해주세요.

## 검토 대상
[파일 경로]

## 특히 확인해야 할 항목
- [ ] N+1 쿼리
- [ ] 불필요한 리렌더링
- [ ] 메모이제이션 필요 여부
- [ ] 번들 크기 영향
- [ ] 데이터베이스 인덱스 활용
- [ ] 캐싱 기회
```

---

## Architecture Review

```
아키텍처 관점에서 코드를 리뷰해주세요.

## 검토 대상
[기능 또는 모듈 설명]

## 현재 구조
[현재 구조 설명]

## 질문
1. 이 구조가 적절한가요?
2. 확장성은 어떤가요?
3. 테스트하기 쉬운가요?
4. 더 나은 대안이 있나요?
```

## PR Review Template

```
PR 리뷰를 요청합니다.

## PR 정보
- Title: [PR 제목]
- Branch: [브랜치명]
- Files changed: [파일 수]

## 변경 요약
[변경 사항 요약]

## 리뷰 포인트
1. [특별히 확인받고 싶은 부분 1]
2. [특별히 확인받고 싶은 부분 2]

## 테스트 방법
[테스트 방법 설명]
```

## Review Response Format

### Issue Severity

| Level | Description | Action |
|-------|-------------|--------|
| Critical | 즉시 수정 필요 | 머지 차단 |
| High | 수정 권장 | 수정 후 머지 |
| Medium | 개선 권장 | 선택적 수정 |
| Low | 사소한 제안 | 정보 제공 |

### Comment Format

```markdown
**[Critical/High/Medium/Low]**: [설명]

**현재 코드:**
```[언어]
[문제 코드]
```

**제안:**
```[언어]
[개선 코드]
```

**이유:**
[개선 이유 설명]
```
